### Render Deployment Steps

#### Option 1: Using render.yaml (Blueprint) - RECOMMENDED

1. **Connect Your Repository**
   - Go to [Render Dashboard](https://dashboard.render.com)
   - Click "New" → "Blueprint"
   - Connect your GitHub repository
   - Render will automatically detect `render.yaml`

2. **Configure Environment Variables**
   
   Render will automatically set most variables from `render.yaml`, but you need to manually set these sensitive values:
   
   ```bash
   ADMIN_PASSWORD=5199152396aA!
   MAIL_USERNAME=futurevisionmachines.com
   MAIL_PASSWORD=icMsHW8b2hU7p2PT
   ```

3. **Deploy**
   - Click "Apply" to create services
   - Render will automatically deploy
   - Database will be created and linked

#### Option 2: Manual Setup

1. **Create PostgreSQL Database**
   - Dashboard → New → PostgreSQL
   - Name: `futurevisionmachines-db`
   - Region: Oregon
   - Plan: Free
   - Database: `fvm`
   - User: `fvm_user`

2. **Create Web Service**
   - Dashboard → New → Web Service
   - Connect repository
   - Name: `futurevisionmachines-website`
   - Region: Oregon
   - Branch: `master`
   - Build Command: `pip install -r requirements.txt`
   - Start Command: `gunicorn 'app:create_app("production")'`

3. **Set Environment Variables**
   
   Go to Environment tab and add:
   
   ```bash
   SECRET_KEY=[Auto-generated by Render]
   FLASK_ENV=production
   DATABASE_URL=[Link to PostgreSQL database]
   ADMIN_PASSWORD=5199152396aA!
   MAIL_SERVER=mail.smtp2go.com
   MAIL_PORT=2525
   MAIL_USERNAME=futurevisionmachines.com
   MAIL_PASSWORD=icMsHW8b2hU7p2PT
   MAIL_DEFAULT_SENDER=info@futurevisionmachines.com
   COMPANY_EMAIL=info@futurevisionmachines.com
   ```

4. **Link Database**
   - In Web Service settings → Environment
   - Click "Add from Database"
   - Select your PostgreSQL database
   - This automatically sets `DATABASE_URL`

### Initialize Database (First Deployment)

After first deployment, run these commands in Render Shell:

```bash
# Access Render Shell from dashboard
flask db init
flask db migrate -m "Initial migration"
flask db upgrade
```

Or use Python shell:
```python
from app import create_app, db
app = create_app('production')
with app.app_context():
    db.create_all()
    print("Database tables created!")
```

### Auto-Deployment

Once connected, **every git push to master** will automatically trigger deployment:

```bash
git add .
git commit -m "Your changes"
git push origin master
```

Render will:
1. Detect the push
2. Run build script
3. Deploy new version
4. No downtime (rolling deployment)

### Troubleshooting

**Database Connection Issues:**
```bash
# Check DATABASE_URL is set correctly
echo $DATABASE_URL

# Test connection
python -c "from app import create_app; app = create_app('production'); print('Connected!')"
```

**Email Not Sending:**
- Verify SMTP2GO credentials in Environment Variables
- Check logs for email errors
- Ensure MAIL_USERNAME and MAIL_PASSWORD are set

**App Won't Start:**
- Check logs for specific error
- Verify all required environment variables are set
- Ensure database migrations are run

### Custom Domain Setup

1. Go to your Web Service → Settings → Custom Domains
2. Add your domain: `futurevisionmachines.com`
3. Add DNS records as instructed by Render
4. Enable "Force HTTPS"
